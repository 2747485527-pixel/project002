# 城市空气质量监测与预测平台介绍

## 1. 项目定位

城市空气质量监测与预测平台是一个面向“城市环境治理、公众服务与应急研判”的数据可视化与智能分析系统。  
平台整合空气质量、气象观测与预测模型，提供从“实时监测”到“趋势分析”再到“风险预警”的一体化能力。

## 2. 核心功能

### 2.1 多层级地图监测

- 支持全国、省、市三级视图联动
- 支持地图点击与排行联动定位目标城市
- 支持空气质量与气象维度的空间分布展示

### 2.2 实时与历史分析

- 实时指标卡：AQI、PM2.5、温度、湿度、风速等
- 历史趋势：按天/月/年查看 PM2.5 等序列变化
- 对比分析：空气与天气指标短期变化排名与详情

### 2.3 预警与研判

- 城市风险评分与污染等级预警
- 极端天气小时级因子输出与前端告警判定
- 生活气象指数（穿衣、紫外线、感冒、洗车等）提示

### 2.4 模型驱动预测

- 支持多种模型版本（树模型、LSTM、Transformer）
- 支持未来 24~168 小时多步预测
- 支持模型版本展示、任务状态查询与结果回看

## 3. 技术创新亮点

### 3.1 多模型并行可替换架构

平台采用统一产物协议，让树模型、LSTM、Transformer 可以并行迭代、快速切换，避免单模型绑定风险。

### 3.2 规则模型与学习模型融合

- 学习模型负责多目标预测（AQI、PM2.5 等）
- 规则模型负责生活指数与极端天气场景解释

这种“可解释规则 + 数据驱动预测”的组合，兼顾准确性与可理解性。

### 3.3 端到端业务闭环

从数据采集、数据库落地、接口聚合到大屏展示与模型联动，形成完整工程闭环，支持快速部署与稳定演示。

### 3.4 高容错数据接入策略

支持多数据源与回退策略，结合离线 CSV 回填机制，降低外部接口波动对系统可用性的影响。

### 3.5 统一接口与联动机制

通过统一 API 协议与城市解析机制，保证多个前端组件在复杂交互场景下一致刷新，减少“数据口径不一致”问题。

## 4. 应用价值

### 4.1 管理部门

- 快速掌握区域空气质量态势
- 辅助污染过程识别与应急响应
- 为治理策略评估提供数据依据

### 4.2 公共服务

- 向公众提供更直观的环境信息
- 提供出行与健康相关的生活气象建议
- 强化高风险天气与污染时段的预警提示

### 4.3 教学与科研

- 作为数据可视化、时序建模、系统工程综合案例
- 支持从数据工程到模型工程的全流程实践

## 5. 系统优势总结

- 功能完整：监测、分析、预测、预警一体化
- 架构清晰：前后端解耦、模型可替换、接口标准化
- 创新突出：多模型融合 + 业务规则解释 + 大屏联动
- 可扩展强：支持新增城市、指标、模型与数据源

## 6. 下一步方向

- 增加模型自动训练与发布流水线
- 引入缓存与指标监控提升服务稳定性
- 强化预测效果评估与模型可解释报告
- 向多源融合与更精细时空预测方向演进
